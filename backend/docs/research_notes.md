# 低碳通勤與台北大眾運輸開放資料查證報告

本報告整理了低碳通勤與遊戲化回饋 App 在計算與資料獲取時的官方來源、係數依據及後續資料對接規劃。

---

## 1. 碳排係數來源查證 (Carbon Factors Sources)

為了確保減碳計算的可信度，我們查閱並比對了台灣官方與國際公開的溫室氣體排放係數：

### 官方來源依據
*   **環境部溫室氣體登錄平台**（前環保署）公佈之《溫室氣體排放係數表》（最新版）：
    *   **機車（Scooter/Motorcycle）**：每人公里約產生 **0.055 kg CO2e**（以常見的 125cc 燃油機車平均油耗折算）。我們將此數值作為 MVP 系統的**比較減碳基準線 (Baseline)**。
    *   **台北捷運（Taipei MRT）**：根據台北捷運公司永續報告書及環境部宣告，台北捷運每人公里碳排放約為 **0.005 - 0.006 kg CO2e**（因大眾運輸載客量大，人均碳排極低，且逐年因綠電比例提升而下降）。我們採用 **0.005 kg CO2e/km**。
    *   **市區公車（City Bus）**：每人公里約產生 **0.030 - 0.038 kg CO2e**（燃油公車與電動公車之加權平均值）。
    *   **步行（Walk）與腳踏車（Bicycle/YouBike）**：直接碳排放均為 **0.0 kg CO2e**。

### 資料庫儲存設計
我們已將這些係數錄入資料庫表 `carbon_factors` 中，便於未來隨政策或係數更新而隨時調整，不寫死於程式碼：
*   `motorcycle` = 0.055
*   `mrt` = 0.005
*   `bus` = 0.030
*   `walk` = 0.000
*   `bike` = 0.000

## 2. 交通部 TDX 與台北交通開放資料對接規劃

為了在第二階段實現更準確的大眾運輸匹配（比對使用者是否真的在搭乘捷運或公車，而非自己開車卻選擇捷運），我們對接了**交通部 TDX (Transportation Data eXchange) 平台**的開放資料。

### 主要對接之 TDX API 端點
1.  **捷運站點資料 (Metro Stations)**：
    *   **API Path**: `/v2/Rail/Metro/Station/TMRT` (台北捷運)
    *   **用途**：獲取捷運各路線之站點名稱、精確經緯度座標。我們已在 MVP 後端手動整理並預載了 25 個台北核心站點座標（如忠孝新生、忠孝復興、忠孝敦化等），後續可改為啟動時自動透過 API 同步更新。
2.  **公車路線與公車站牌資料 (City Bus)**：
    *   **API Path**: `/v2/Bus/Station/City/Taipei`
    *   **用途**：取得台北市所有公車站牌座標與路線關聯。可用於掃描使用者 GPS 是否持續沿著公車軌跡與站牌點位移動。
3.  **YouBike 即時站點資料 (Bicycle Stations)**：
    *   **API Path**: `/v2/Bike/Availability/City/Taipei`
    *   **用途**：獲得台北市 YouBike 租借站位置及車輛數。可判定使用者起訖點是否在 YouBike 站點 50 米範圍內，輔助判定騎乘行為。

### 自動匹配算法架構設計
當使用者上傳一段軌跡並結束旅程時，後端可透過以下方式利用 TDX 資料：
*   **起訖點 Buffer 匹配**：起訖點是否在捷運站 300 公尺 Bounding Box 內。
*   **斷訊區間與路線軌跡重合度**：若是捷運，其地下路段會有顯著的 GPS 遺失，但起訖時間恰好符合該兩站之間的捷運行駛班距（可透過 TDX ODF 站間行駛時間 API 比對）。
*   **道路站牌距離**：如果軌跡在道路上且每逢公車站牌就出現停等或減速，則可藉由與公車路線軌跡的相似度（Frechet Distance 演算法）判定為公車乘客。
