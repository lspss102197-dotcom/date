# 四週甘特圖

本甘特圖提供 PM 快速掌握前端工程師、後端工程師、設計師與 PM 在 MVP 前四週的工作分配。詳細任務請看各角色資料夾內的 `TASK.md`。

## 週次分工表

| 週次 | PM | 設計師 | 前端工程師 | 後端工程師 |
| --- | --- | --- | --- | --- |
| 第 1 週 | 確認 MVP 範圍、驗收標準、風險清單 | 使用者流程、畫面清單、旅程流程 | Flutter 骨架、路由、登入註冊 UI | FastAPI 骨架、PostgreSQL、Auth |
| 第 2 週 | 追蹤設計與 API 是否對齊 | Wireframe、資訊架構、錯誤狀態 | GPS 權限、開始與結束旅程、GPS 上傳 | Trips / GPS Points API |
| 第 3 週 | 安排整合測試計畫 | 高保真、Design System、視覺回饋 | Google Map、路線顯示、旅程結果 | 距離時間計算、碳排與減碳計算 |
| 第 4 週 | 驗收 MVP、整理下一階段需求 | 狀態補齊、資產交付、協助驗收 | API 整合、實機測試、視覺回饋 | 刪除旅程、統計 API、測試與文件 |

## PM 驗收重點

- 第 1 週結束：三方都理解 MVP 範圍，登入與旅程流程沒有重大分歧。
- 第 2 週結束：前後端能以 mock 或初版 API 串起旅程建立與 GPS 點上傳。
- 第 3 週結束：使用者可以完成一次旅程並看到初步結果。
- 第 4 週結束：MVP 可展示登入、旅程紀錄、地圖、結果、刪除與最小視覺回饋。
